    **Step A: Add custom output format and media type to `hugo.toml`**

    Add the following block to the end of `hugo.toml` (before any existing closing content). The `[mediaTypes]` and `[outputFormats]` blocks are new; the `[outputs]` block adds the custom format to the home page output.

    In Hugo, custom output formats are defined in the site config. The MediaType determines the content type, the OutputFormat determines which template Hugo looks for and what file it produces.

    ```toml
    # Custom media type for posts index JSON
    [mediaTypes]
      [mediaTypes."application/postsindex"]
        suffixes = ["postsindex.json"]

    # Custom output format using the PostsIndex media type
    [outputFormats]
      [outputFormats.PostsIndex]
        mediaType = "application/postsindex"
        baseName = "posts-index"
        isPlainText = true
        notAlternative = true

    # Add PostsIndex output to home page — this is where .Data.Pages iterates all posts
    [outputs]
      home = ["HTML", "PostsIndex", "RSS"]
    ```

    IMPORTANT: Do NOT remove or modify any existing keys in hugo.toml. Only add these new top-level blocks. The existing `[params]`, `[taxonomies]`, `[[menu.main]]` entries must remain untouched.

    **Step B: Create `layouts/_default/index.postsindex.json`**

    Create the directory and file. This is a Hugo template that generates JSON. Hugo's `jsonify` function converts Go data structures to JSON strings.

    Create file at: `layouts/_default/index.postsindex.json`

    ```json
    {{- /* Hugo template: posts-index.json — generates compact post index for AI system prompt */ -}}
    {{- $posts := slice -}}
    {{- $baseURL := .Site.BaseURL -}}
    {{- range where .Data.Pages "Section" "posts" -}}
      {{- if not .Draft -}}
        {{- $post := dict
          "title" .Title
          "description" .Description
          "summary" .Summary
          "date" (.Date.Format "2006-01-02")
          "link" (.RelPermalink | absURL)
          "tags" .Params.tags
          "categories" .Params.categories
        -}}
        {{- $posts = $posts | append $post -}}
      {{- end -}}
    {{- end -}}
    {{- $posts | jsonify -}}
    ```

    Template logic explanation:
    - `range where .Data.Pages "Section" "posts"` — iterates ALL pages where Section="posts" (the posts list page)
    - `if not .Draft` — excludes drafts (matching RAG-01 AC "Draft posts (draft: true) are excluded")
    - `dict` builds a Go map with the 7 fields required by RAG-01
    - `$posts | jsonify` converts the Go slice of maps to a JSON array
    - `RelPermalink | absURL` generates full URLs like `https://2p1c.life/posts/some-post/`

    ```javascript
    import fs from 'node:fs';
    import path from 'node:path';

    // Simulate loadPostsIndex (same logic as server.js)
    const indexPath = path.resolve('/Users/zyt/ANW/2p1c_blog/public/posts-index.json');
    const raw = fs.readFileSync(indexPath, 'utf-8');
    const posts = JSON.parse(raw);

    console.log(`=== Posts Index Loaded: ${posts.length} posts ===`);

    // Check required fields
    const requiredFields = ['title', 'description', 'summary', 'date', 'link', 'tags', 'categories'];
    for (const [i, post] of posts.entries()) {
      for (const field of requiredFields) {
        if (!(field in post)) {
          console.error(`Post ${i} ("${post.title || 'UNKNOWN'}") missing field: ${field}`);
          process.exit(1);
        }
      }
    }

    // Check draft exclusion: verify no post has "draft: true" equivalent
    // (Hugo template already excludes them, but verify no residual)
    const draftKeywords = posts.filter(p => p.title.includes('[DRAFT]') || p.title.includes('draft'));
    if (draftKeywords.length > 0) {
      console.warn('Warning: posts with draft-like titles found:', draftKeywords.map(p => p.title));
    }

    // Format as system prompt block (same logic as loadPostsIndex in server.js)
    const lines = posts.map(p => {
      const tags = (p.tags || []).join(', ');
      const date = (p.date || '').slice(0, 10);
      return `- ${p.title} (${date}) [${tags}] ${p.link}`;
    });
    const block = lines.join('\n');

    // Token estimate (same logic as plan 01-01 estimateTokens)
    const estimatedTokens = Math.ceil(block.length / 3);
    console.log(`Formatted block: ${lines.length} lines, ~${estimatedTokens} estimated tokens`);

    if (estimatedTokens > 2500) {
      console.error(`TOKEN BUDGET EXCEEDED: ${estimatedTokens} > 2500. Posts index too large.`);
      process.exit(1);
    }

    // Verify content searchability: check that specific tag lookups would work
    const allTags = new Set(posts.flatMap(p => p.tags || []));
    console.log(`All tags: ${[...allTags].join(', ')}`);

    // Verify Chinese characters are preserved (sample first 3 titles)
    console.log('\nSample posts:');
    for (const p of posts.slice(0, 5)) {
      console.log(`  ${p.title} | ${p.date} | tags: [${(p.tags||[]).join(', ')}]`);
    }

    console.log('\n=== RAG Integration Verification PASSED ===');
    console.log(`Posts: ${posts.length}, Tokens: ${estimatedTokens}/2500, Tags: ${allTags.size}`);
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hugo template → public/posts-index.json | Template processes user-authored Markdown front matter — must not execute embedded code |
| public/posts-index.json → Node.js server | Static JSON file loaded at server startup — validate structure before inserting into prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | posts-index.json content | accept | File is generated at Hugo build time, not user-writable in production. If an attacker gains server access to overwrite the file, they already have full system access. |
| T-02-02 | Information Disclosure | Post summaries in system prompt | accept | All post data in the index is already publicly published on the blog. The summary field is a subset of what's already public — no new information exposed. |
| T-02-03 | Elevation of Privilege | Hugo template code execution | mitigate | Hugo templates are sandboxed Go templates — they cannot execute arbitrary code. The `jsonify` function only serializes data; it cannot inject executable content. Verify template uses only Hugo-provided functions (dict, range, where, jsonify). |
</threat_model>
