## States (Interaction Contract)

### 1. Personality Selector Dropdown (PRS-01)

**Location:** `.ai-chat-header`, between header title and actions area (new slot).

**Element:** Native `<select>` dropdown, styled to match vintage theme.

| State | Visual | Interaction |
|-------|--------|-------------|
| Default | Transparent bg, white text, border `rgba(255,255,255,0.25)`, font 13px | Click to open native option list |
| Hover | bg `rgba(255,255,255,0.2)`, border `rgba(255,255,255,0.4)` | pointer cursor |
| Focus | Outline: `2px solid var(--highlight)`, outline-offset: `1px` | Keyboard arrows navigate options |
| Open | Native OS dropdown list (unstyled -- native `<select>`) | `mouseleave` on panel must NOT close panel while dropdown is open |
| Disabled | `opacity: 0.5`, `cursor: not-allowed` | No interaction (during streaming) |

**Dropdown options:**
1. 温暖学长 (default, `warm-senior`)
2. 幽默朋友 (`humorous-friend`)
3. 文艺青年 (`literary-youth`)
4. 重金属黑色死亡摇滚青年 (`metal-rock-youth`)

**Critical interaction constraint:** The chat panel's `mouseleave` handler must detect whether the native `<select>` dropdown is open. Use a `mousedown`/`focus`/`blur` tracking flag on the select element to suppress `closePanel()` during dropdown interaction.

**DOM structure:**
```html
<div class="ai-chat-persona">
  <label for="ai-chat-persona-select" class="sr-only">选择 AI 性格</label>
  <select id="ai-chat-persona-select" class="ai-chat-persona-select">
    <option value="warm-senior">温暖学长</option>
    <option value="humorous-friend">幽默朋友</option>
    <option value="literary-youth">文艺青年</option>
    <option value="metal-rock-youth">重金属黑色死亡摇滚青年</option>
  </select>
</div>
```

**CSS contract (new class `.ai-chat-persona-select`):**
```css
.ai-chat-persona-select {
    background: rgba(255, 255, 255, 0.1);
    border: 1px solid rgba(255, 255, 255, 0.25);
    border-radius: var(--radius-sm);  /* 4px */
    color: var(--window-header-color);  /* #ffffff */
    font-family: inherit;
    font-size: 13px;
    padding: 4px 8px;
    cursor: pointer;
    max-width: 120px;
    text-overflow: ellipsis;
}
.ai-chat-persona-select:hover {
    background: rgba(255, 255, 255, 0.2);
    border-color: rgba(255, 255, 255, 0.4);
}
.ai-chat-persona-select:focus-visible {
    outline: 2px solid var(--highlight);
    outline-offset: 1px;
}
```

**Mobile (320px):** `max-width` reduces to `100px`; font-size stays 13px. The dropdown must not push the close button off-screen. Use `gap: 4px` between header elements at 320px.

### 2. Inline Greeting UI (GRT-01)

**Location:** Inside `#ai-chat-messages`, as the first visible element (replacing `.ai-chat-welcome` for new users).

**Element:** Greeting message bubble + inline form with text input and submit button, styled identically to the existing chat composer.

| State | Visual | Interaction |
|-------|--------|-------------|
| Default | Assistant message bubble with greeting text, name input below matching `#ai-chat-input` styling, submit button matching `.btn-send` | Focus auto-set to name input on render |
| Input focus | `box-shadow: 0 0 0 3px var(--chat-focus-ring)`, border-color `var(--highlight)` | Keyboard: Enter submits |
| Input filled | Submit button enabled (solid blue gradient) | Click/tap submits |
| Input empty | Submit button at 50% opacity (disabled state) | Cannot submit |
| Submitting | Input and button disabled, button shows loading indicator | `POST /chat/register` in progress |
| Success | Greeting UI fades out (200ms), normal chat UI fades in | Auto-transition to chat mode |
| Error | Error text in red (`#ef4444`) below input | Input re-enabled, user can retry |

**DOM structure:**
```html
<div id="ai-chat-greeting" class="ai-chat-greeting" role="status">
  <!-- Greeting message bubble (styled as ai-chat-msg-assistant) -->
  <div class="ai-chat-msg ai-chat-msg-assistant">
    <div class="ai-chat-msg-avatar">
      <img src="[ai-avatar]" alt="AI avatar" />
    </div>
    <div class="ai-chat-msg-body">
      你好！我是这个博客的 AI 助手。在开始之前，能告诉我你的名字吗？
    </div>
  </div>
  <!-- Name input form -->
  <form id="ai-chat-greeting-form" class="ai-chat-greeting-form">
    <div class="ai-chat-input-wrapper">
      <label for="ai-chat-greeting-input" class="sr-only">输入你的名字</label>
      <input
        id="ai-chat-greeting-input"
        type="text"
        placeholder="输入你的名字..."
        maxlength="50"
        autocomplete="given-name"
      />
    </div>
    <button id="ai-chat-greeting-submit" type="submit" class="btn-send" disabled>
      确认
    </button>
  </form>
  <p id="ai-chat-greeting-error" class="ai-chat-greeting-error" hidden></p>
</div>
```

**CSS contract (new classes):**

### 3. Chat Panel Pin During Greeting (GRT-01)

**Interaction contract:** When `.ai-chat-greeting` is visible in the messages area, the chat panel's `mouseleave` close behavior is disabled. The panel stays pinned open regardless of mouse position.

**Implementation note:** Backend of `initAiChatWidget()` in `main.js` -- add a check function `isGreetingActive()` that returns `true` when `#ai-chat-greeting` is present in the DOM. Inside the `panel.addEventListener('mouseleave', ...)` handler, early-return if greeting is active.

**Pin release:** When greeting completes (name registered successfully) and `.ai-chat-greeting` is removed from the DOM, `mouseleave` close behavior resumes normally.

### 4. Existing States (Regression Contract)

All existing chat states and interactions must continue functioning unchanged:

| State | Element | Contract |
|-------|---------|----------|
| Panel open | `#ai-chat-panel` | Toggle hover/click opens, `aria-expanded="true"` |
| Panel closed | `#ai-chat-panel` | `mouseleave` closes (except during greeting), `aria-expanded="false"`, `hidden` attribute |
| Streaming | Messages area | Input disabled, send button disabled, clear disabled, thinking animation visible, dropdown disabled |
| Idle | Messages area | All controls enabled, input focused |
| API offline | Status area | Dot turns `offline` class, text changes to "离线" |
| Input at 90% | Char counter | `.warning` class applied (orange `#f59e0b`) |
| Input at 100% | Char counter | `.error` class applied (red, via `--chat-notification`) |
| Clear success | Messages area | All messages removed, "new session!" message shown |
| Suggestion click | Input | Text pre-filled, welcome removed, input focused |

**Source:** Existing JS behavior in `main.js` lines 438-702, CSS states in `main.css` lines 1347-2056.

## Layout Contract

**Focal point:** The AI greeting message bubble (new users) / latest assistant message (returning users) is the primary visual anchor — positioned top-left in the messages area with background contrast against the dark messages container.

**Visual hierarchy:** 1. AI greeting/message bubble, 2. Name input field (greeting) or chat input (normal), 3. Submit/send button, 4. Chat header with persona selector.

### Chat Header Layout (with Personality Dropdown)

```
[Avatar] [AI 助手] [▼ 温暖学长 ▼]  ...flex-gap...  [🗑 Clear] [✕ Close]
          [在线 ●]

|<--------- header-title --------->|  <persona>  |<------- actions ------->|
```

- Header uses `display: flex; justify-content: space-between; align-items: center; gap: 8px;`
- `.ai-chat-persona` sits BETWEEN `.ai-chat-header-title` and `.ai-chat-actions`
- At 320px: gap reduces to 4px, persona `max-width: 100px`, all three sections fit on one row
- The dropdown text may truncate with `text-overflow: ellipsis` at narrow widths

### Greeting Flow Layout

```
┌─────────────────────────────────┐
│ [Header with persona dropdown]  │
├─────────────────────────────────┤
│                                 │
│  ┌──────────────────────┐       │
│  │ AI avatar + bubble:  │       │
│  │ "你好！我是这个博客  │       │
│  │  的AI助手..."        │       │
│  └──────────────────────┘       │
│                                 │
│  [  输入你的名字...   ] [确认]  │
│                                 │
├─────────────────────────────────┤
│ (normal composer hidden)        │
└─────────────────────────────────┘
```

- During greeting, the normal `#ai-chat-form` is hidden (`display: none`)
- Greeting form sits inside `#ai-chat-messages` as its own element
- After registration success: greeting elements fade out, normal `#ai-chat-form` shows, welcome message shows for a brief moment then transitions to ready state
