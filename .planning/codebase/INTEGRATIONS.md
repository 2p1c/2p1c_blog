# Codebase Integrations

Generated: 2026-04-25

## Overview

The project has a small set of integrations: Hugo/GitHub Pages-style static publishing, a DeepSeek-compatible chat completions upstream, local SQLite persistence, browser `localStorage`, Google Analytics configuration, and server deployment over SSH/rsync.

## AI Chat Upstream

`ai-assistant/server.js` sends streaming chat requests to:

- Base URL: `DEEPSEEK_BASE_URL`, default `https://api.deepseek.com/v1`.
- Endpoint: `${DEEPSEEK_BASE_URL}/chat/completions`.
- Auth header: `Authorization: Bearer ${DEEPSEEK_API_KEY}`.
- Model: `DEEPSEEK_MODEL`, default `deepseek-chat`.
- Streaming protocol: upstream SSE-style chunks with `data:` payloads.

The backend translates upstream deltas into local SSE events:

- `event: token` with `{ "delta": "..." }`.
- `event: done` with the final reply and session id.
- `event: error` for upstream or parsing failures.

## SQLite

The chat backend uses SQLite through `better-sqlite3` in `ai-assistant/server.js`.

Database path:

- `DB_PATH` if provided.
- Otherwise `./data/chat.db`, resolved relative to `ai-assistant/`.

Tables are created at startup:

- `sessions(id, created_at, updated_at)`.
- `messages(id, session_id, role, content, created_at)`.

The backend persists:

- User messages before calling the upstream model.
- Assistant messages after the full streamed reply is assembled.
- Session updates during message insert and clear operations.

## Browser Storage

The theme chat widget in `themes/vintage-web-hugo-theme/assets/js/main.js` stores a session id in `localStorage` under `ai_chat_session_id`.

The standalone backend chat page generated by `renderChatPage()` in `ai-assistant/server.js` uses a different key, `chat_session_id`.

## Site Chat Widget

The Hugo site enables chat through `[params.ai_chat]` in `hugo.toml`:

- `enabled = true`.
- `api_base = "/api/chat"`.
- `launcher_image = "/images/chat-launcher.png"`.
- `ai_avatar = "/images/ai-avatar.jpg"`.

`themes/vintage-web-hugo-theme/layouts/partials/ai-chat.html` renders the widget shell and places these values into `data-*` attributes. `themes/vintage-web-hugo-theme/assets/js/main.js` reads those attributes, streams from `${apiBase}/stream`, clears via `${apiBase}/clear`, and checks health from a derived `${apiBase}/health` path.

For localhost, `resolveAiChatApiBase()` maps the default `/api/chat` base to `http://127.0.0.1:4310/chat`.

## Reverse Proxy

`ai-assistant/README.md` documents Nginx routes:

- `/api/chat/stream` to `http://127.0.0.1:3001/chat/stream`.
- `/api/chat/clear` to `http://127.0.0.1:3001/chat/clear`.
- `/api/health` to `http://127.0.0.1:3001/health`.

The API contract in `API_CONTRACT.md` uses the public `/api/chat/*` namespace and calls out that streaming routes need buffering disabled.

## Analytics and Social Links

`hugo.toml` configures:

- `google_analytics = "G-XXXXXXXXXX"`.
- Twitter/X, GitHub, email, and phone values under `[params.social]`.

`themes/vintage-web-hugo-theme/layouts/_default/baseof.html` renders social links and includes Hugo's internal Google Analytics template.
